# Барбершоп «Сайтама»

CMS-система для онлайн-записи в барбершоп: каталог мастеров и услуг, бронирование с защитой от двойной записи, личные кабинеты клиента/мастера/администратора, отзывы, статистика.

## Стек

**Backend:** FastAPI · SQLAlchemy 2.0 (`Mapped`/`mapped_column`) · Pydantic v2 (`Annotated`) · PostgreSQL · Alembic · JWT (`python-jose`) · bcrypt

**Frontend:** Vue 3 (`<script setup>`) · Pinia · Vue Router · Tailwind CSS · Heroicons · Vite

## Возможности

- **Аутентификация и роли** — регистрация/вход по email+паролю (bcrypt), JWT, три роли (`client` / `master` / `admin`), восстановление пароля по ссылке (в дев-режиме ссылка пишется в лог сервера — SMTP не подключён)
- **Каталог и бронирование** — поиск/фильтр/сортировка мастеров и услуг, расчёт свободных слотов по расписанию мастера, атомарная защита от двойного бронирования (проверка на уровне приложения **и** ограничение `EXCLUDE USING gist` на уровне БД)
- **Жизненный цикл записи** — явная машина состояний (`pending → confirmed → done`, отмена из `pending`/`confirmed`), мастер подтверждает/завершает запись, клиент отменяет свою
- **Отзывы** — клиент оставляет отзыв на завершённую запись, админ публикует/скрывает
- **Личный кабинет мастера** — свои записи со сменой статуса, редактирование расписания по дням недели, свои отзывы
- **Админ-панель** — статистика (счётчики + график регистраций), управление пользователями (роли, мягкое удаление), услугами (CRUD, мягкое удаление), мастерами (фото, привязка услуг), модерация отзывов
- **Мягкое удаление** пользователей/мастеров/услуг — историю записей не теряем при удалении
- Пагинация, инлайн-валидация форм, toast-уведомления, адаптивная вёрстка (375 / 768 / 1024+)

Подробное описание архитектуры и условностей кода — в [`CLAUDE.md`](./CLAUDE.md). Дизайн-система фронтенда — в [`frontend/design-system/сайтама-barbershop/MASTER.md`](./frontend/design-system/сайтама-barbershop/MASTER.md).

```
backend/    FastAPI-приложение (routes → services → repositories → models)

```

## Требования

- Python 3.14 (использовался при разработке; не тестировалось на более старых версиях)
- Node.js 18+ (минимум для Vite 5)
- PostgreSQL 13+ (нужны права на `CREATE EXTENSION` — для `btree_gist`, см. ниже)

## Запуск backend

```bash
cd backend
python -m venv venv && source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Создайте `backend/.env`:

```env
DATABASE_URL=postgresql://user:password@localhost:5432/hair_salon
SECRET_KEY=<сгенерируйте случайную строку>
```

(полный список переменных и их значения по умолчанию — в `backend/app/config.py`)

Создайте базу данных, затем накатите миграции (миграция сама создаст расширение `btree_gist`, ENUM-типы и `EXCLUDE`-ограничение — нужны права на создание расширений):

```bash
alembic upgrade head
python run.py     # http://localhost:8000, Swagger-документация на /api/docs
```

## Запуск frontend

```bash
cd frontend
npm install
npm run dev       # http://localhost:5173, проксирует /api на localhost:8000
```

## Известные ограничения

OAuth-вход и реальная отправка email (восстановление пароля) не реализованы — задача не предполагала подключение внешнего провайдера. Деплой и CI/CD не настроены.
