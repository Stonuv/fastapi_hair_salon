## Global Rules

### Color Palette

Coffee-brown + warm gold, per the prepared reference notes. Adapted from the raw "Coffee Shop" palette
result for practical use across dense dashboard surfaces (the raw `#FEF3C7` cream is too saturated as a
*global* page background at scale — reserved for accent surfaces instead).

| Role | Hex | Tailwind token | Usage |
|------|-----|-----------------|-------|
| Primary (brown) | `#78350F` | `brand-900` | Sidebar/header bg, primary buttons, active nav state |
| Secondary (warm brown) | `#92400E` | `brand-800` | Hover states, secondary accents |
| Brand mid | `#B45309` | `brand-700` | Links, icon accents |
| CTA/Accent (gold) | `#FBBF24` | `accent-400` | The *one* highest-priority action per page (book, confirm) |
| Accent surface (cream) | `#FEF3C7` | `accent-100` | Badges, highlighted rows, callout cards — not full-page bg |
| Page background | `#FBF8F3` | `stone-50` | Default page canvas (very light warm neutral, not full cream) |
| Card surface | `#FFFFFF` | `white` | Cards/panels sitting on the page background |
| Text (headings) | `#2B1607` | `ink-900` | Headings — dark warm brown, not pure black |
| Text (body) | `#52453B` | `ink-600` | Body copy — passes 4.5:1 on `stone-50`/white |

### Typography

Raw tool pick (Fira Code/Fira Sans, Noto Serif/Sans JP) rejected: wrong mood and no real reason to force
Japanese font coverage on a Russian-language app. Both fonts below have full Cyrillic character sets.

- **Heading / brand font:** Playfair Display (elegant serif, used for page H1s, the logo wordmark, hero
  text on the public Masters catalog — sparingly, not on dense dashboard chrome)
- **UI / body font:** Inter (everything else — nav, buttons, forms, tables, dashboard headers, body copy)
- **Mood:** warm, classic-barbershop, confident — not playful, not corporate-cold

**CSS Import:**
```css
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Playfair+Display:wght@600;700&display=swap');
```

**Tailwind:** `fontFamily: { display: ['Playfair Display', 'serif'], sans: ['Inter', 'sans-serif'] }`

### Spacing / Radius / Shadows

Use Tailwind defaults directly (`p-4`, `gap-6`, `rounded-xl`, `shadow-md`, etc.) — no custom spacing scale
needed on top of Tailwind's. Standard radius for cards/inputs: `rounded-xl` (12px). Buttons: `rounded-lg` (8px).

### Icons

Heroicons (`@heroicons/vue`, outline for nav/secondary, solid for filled/active states) — **never emoji as
UI icons**. Fixed `w-5 h-5` for inline icons, `w-6 h-6` for nav.

## Layout Patterns (per page type — this app has 3, not 1)

1. **Public/marketing-ish** (Masters catalog, Login/Register): centered content, generous whitespace,
   `Playfair Display` for the hero/page title, card grid for master listings. Top nav (`AppHeader`), no sidebar.
2. **Client app pages** (Booking, My Appointments/Profile, Reviews): top nav + content, card-based lists,
   not a sidebar — these are occasional-use single-task flows, a persistent sidebar would be overkill.
3. **Dashboard pages** (Master Dashboard, Admin Panel): **sidebar + main content**, per the prepared
   reference screenshots — sidebar nav (icon + label, active state = brand-900 bg), top bar with page title
   + user menu, main area uses card/table grids. Admin gets a stats row (KPI cards) above the fold per
   `description.txt`'s mapping (calendar → schedule widget, "online users" list → today's appointments,
   course cards → appointment/review cards with a status pill + quick action).

## Component Specs

- **Buttons:** `btn-primary` = `bg-brand-900 text-white hover:bg-brand-800`; `btn-accent` (the one CTA per
  page, e.g. "Записаться") = `bg-accent-400 text-ink-900 hover:brightness-95`; `btn-ghost` = transparent +
  border, brand-900 text. All: `rounded-lg px-4 py-2.5 font-medium transition-colors duration-200
  cursor-pointer disabled:opacity-50 disabled:cursor-not-allowed`. Loading state disables the button and
  swaps label for a small spinner (never just freezes).
- **Cards:** `bg-white rounded-xl shadow-sm border border-stone-200 p-6`, hover (when clickable):
  `hover:shadow-md transition-shadow duration-200` — no transform/scale on hover (avoids layout shift).
- **Inputs:** `rounded-lg border border-stone-200 px-3.5 py-2.5 text-base focus:outline-none
  focus:ring-2 focus:ring-brand-900/30 focus:border-brand-900`. Always paired with a real `<label>`.
  Inline error text in `text-danger` below the field, field border turns `border-danger` on error.
- **Status pills** (appointment/review state): rounded-full px-2.5 py-0.5 text-xs font-medium,
  color-coded (pending=amber, confirmed=blue, done=success green, cancelled=danger red).
- **Toasts:** bottom-right stack, auto-dismiss after 4s, manually dismissible, icon + message,
  success/error/info variants — satisfies the spec's "≤5s auto-dismiss" requirement.
- **Skeletons:** `animate-pulse bg-stone-200 rounded` placeholders for cards/table rows while loading —
  never a blank screen during a fetch.
- **z-index scale:** `z-10` (sticky table headers) / `z-20` (dropdowns) / `z-30` (sidebar on mobile overlay)
  / `z-40` (toasts) / `z-50` (modals). Never arbitrary values.

## Forms & Validation

No external form library (kept off the dependency list deliberately — this app's forms are simple enough
that a tiny shared composable covers it). Validate on blur + on submit; show the first error inline under
the field; disable submit while a request is in flight; show a toast on server-side validation failure
(rubric requires both client *and* server validation — the backend already enforces everything server-side,
the frontend adds the inline pre-submit layer).

## Anti-Patterns (Do NOT Use)

- ❌ Emojis as icons
- ❌ Forcing the raw tool's "Exaggerated Minimalism" oversized-type landing pattern onto dashboard pages
- ❌ A persistent sidebar on the public catalog/booking flow (wrong pattern for an occasional-use task)
- ❌ Pure-cream (`#FEF3C7`) as a full-page background — reserve for accent surfaces only
- ❌ Scale-transform hovers on cards (causes layout shift) — use shadow/border-color transitions instead
- ❌ Validating only on submit
- ❌ Persistent toasts that never auto-dismiss

## Pre-Delivery Checklist

- [ ] No emojis used as icons (Heroicons only)
- [ ] `cursor-pointer` on all clickable elements, visible hover + focus states
- [ ] Light-mode text contrast 4.5:1 minimum (use `ink-900`/`ink-600`, not low-contrast grays)
- [ ] Responsive at 375px / 768px / 1024px / 1440px, no horizontal scroll
- [ ] Forms: real `<label>`s, inline errors, disabled-while-submitting buttons
- [ ] Toasts auto-dismiss ≤5s
- [ ] `prefers-reduced-motion` respected on any non-essential animation
